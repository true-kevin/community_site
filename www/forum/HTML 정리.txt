열고 닫음을 동시에 하는 태그들: <hr />, <br />

<html lang="en-US">, <html lang="ko"> 언어 설정을 해줌으로 스크린리더나 서치엔진의 해석을 돕는다.

<p title="I'm a tooltip"> title 어트리뷰트는 툴팁을 만들어준다.

href 사용 태그들: <a>, <link>, <base>, <link>
src 사용 태그들: <img>, <iframe>, <script>

<img> 태그에서 alt 어트리뷰트는 사진의 대체 문자가 들어간다.(스크린리더나 x박 처리를 위해)
width, height는 무조건 넣어주는 것이 좋다. 페이지가 깜박거리는 것을 방지할 수 있다.


<h1> ~ <h6> 태그들은 주제를 나타내는 주제다. 큰 주제 부터 작은 주제로 정리하는 용도로 사용하여야 한다.(계층 구조를 갖게 됨)

<hr> 태그는 주제나 내용를 분리해주는 용도로 사용된다.

css 적용 방법은 '인라인', '인터널', '익스터널'로 3가지 방식이 있다.
인라인: <h1 style="color:blue;"> 어트리뷰트를 이용한다.
인터널: <style></style> 태그를 이용한다.
익스터널: <link rel="stylesheet" href="styles.css"> 다음 방법으로 인클루드한다.


각 태그를 의미에 맞게 사용해야 한다
<b> - Bold text
<strong> - Important text
<i> - Italic text
<em> - Emphasized text
<mark> - Marked text
<small> - Small text
<del> - Deleted text
<ins> - Inserted text(밑줄)
<sub> - Subscript text
<sup> - Superscript text

<q> 태그를 통해서 짧은 인용절을 표현할 수 있다.
<blockquote cite="http://www.worldwildlife.org/who/index.html"> 태그를 통해서 인용 사이트를 표기할 수 있다.(cite 어트리뷰트)

<cite>The Scream</cite> 태그는 작품 명을 쓸 때 사용한다.

<abbr title="World Health Organization">WHO</abbr> 태그는 줄임말이나 두음문자를 나타낼 때 사용한다. title 어튜리뷰트로 원본 단어를 알 수 있다.

<address> 태그를 통해 연락가능한 정보를 기입한다.(이텔릭으로 표현)

<bdo dir="rtl"> 태그는 Bi-Directional Override란 의미고 dir="rtl"은 direction=right to left란 의미로 내용을 거꾸로 출력한다.

<code>: 프로그래밍 코드 표현을 위한 태그다.
<samp>: 프로그램 출력을 표기하기 위한 태그다.
<kbd>: 키보드 인풋을 표기한다.

<!-- 내용 --> 주석을 나타내는 태그다. 앞에만 !(느낌표)가 있다는 것을 알아두자.

색은 rgb로 표현된다. 빛의 삼원색 색이 많이 섞일 수록 밝아지고 흰색이 된다. 255,255,255 or ffffff 이 둘은 검은색이다.

<a> 태그는 하이퍼링크를 나타낸다.
a:link    {color:green; background-color:transparent; text-decoration:none} // 기본색
a:visited {color:pink; background-color:transparent; text-decoration:none} // 방문 후 색
a:hover   {color:red; background-color:transparent; text-decoration:underline} // 마우스 오버
a:active  {color:yellow; background-color:transparent; text-decoration:underline} // 클릭한 순간
위처럼 내가지 경우의 색을 조정할 수 있다.
text-decoration: none;을 통해 밑줄을 없앨 수 있다.
주의! 'a:link' 이 표현 방식을 잘 기억하자. :(콜론)을 이용하고 있다.
또한 링크가 열리는 타겟을 5가지 방식으로 표현한다.
_blank - Opens the linked document in a new window or tab
_self - Opens the linked document in the same window/tab as it was clicked (this is default)
_parent - Opens the linked document in the parent frame
_top - Opens the linked document in the full body of the window (현재 페이지 최상위 조상 창에서 연다.)
framename - Opens the linked document in a named frame
<a href="#id"> 이 방식으로 해당 id의 태그로 이동할 수 있다. (북마크 기능)

html에서의 경로
html 코드에서의 루트는 항상 웹서버의 'www' 폴더이다. 그 상위 폴더에는 접근할 수 없다.
절대 경로와 상대 경로 그리고 도메인 표기 방식을 사용할 수 있다.
절대 경로: '/'로 시작한다. 예) /images/html5.gif
상대 경로: './파일명' 또는 '파일명'으로 표기한다. 예) ./html5.gif or html5.gif
도메인표기: 그냥 도메인을 포함한 url을 써준다. 예) http://www.w3schools.com/images/w3schools_green.jpg ('http://' 잊지 말자)

테이블에서 보더를 한줄로 접는 방법
table, tr, th, td {
    border: 1px solid black;
    border-collapse: collapse;
}
위와 같은 css를 사용하자
table {
    border-spacing: 15px;
}
위와 같은 css를 통해 보더간의 간격을 설정할 수 있다. (단, collapse를 했을 경우 무효화 된다.)
중요! <th colspan="2">은 가로 병합, <th rowspan="2">은 세로 병합이다.

<ul>: 순서 없는 리스트
<ol>: 순서 있는 리스트
각 리스트 노드는 <li>로 표현된다.
<dl>: 디스크립션 리스트로서 '<dt> 텀'과 '<dd> 노드'로 표현 된다.
<dl>
  <dt>Coffee</dt>
  <dd>- black hot drink</dd>
  <dt>Milk</dt>
  <dd>- white cold drink</dd>
</dl>

블록 구분
블록 레벨 엘리먼트:
1. 가능한 가장 큰 width를 차지한다.
2. 앞과 뒤에 각각 한번씩 줄넘김을 한다.
3. width, height 값을 가질 수 있다.
예) <div>, <h1>~<h6>, <p>, <form>
인라인 엘리먼트:
1. 가능한 가장 작은 width를 차지한다. (엘리먼트 내부 요소에 맞춘다.)
2. 줄넘김을 하지 않고 위치한다.
3. width, height 값을 가질 수 없다.
예) <span>, <a>, <img>

<div>는 주로 엘리먼트의 집합을 묵어줄 때 사용된다.
<span>은 텍스트를 묶어줄 때 사용된다.

<iframe> 태그를 통해 다른 웹페이지를 보여줄 수 있다.
예) <iframe src="demo_iframe.htm" height="200" width="300"></iframe> 
<a> 태그의 링크 타겟이 될 수 있다.
예)
<iframe src="demo_iframe.htm" name="iframe_a"></iframe>
<p><a href="http://www.w3schools.com" target="iframe_a">W3Schools.com</a></p>
단, <iframe> 태그는 반드시 name이 있어야 한다.(그래야 target으로 삼으니까)

<head>는 메타데이터(데이터 즉, html 문서에 대한 데이터)를 닮기 위한 태그이다.
<title>제목<title> // html 문서의 제목, 브라우저의 텝에 표현된다. (서치엔진 입장에서 중요한 정보이다.)
<link rel="stylesheet" href="mystyle.css"> css를 가져온다.
<style> css 표기
<script> javascript 표기
*. <meta> 태그
캐릭터셋, 리프레쉬 타임, 키워드, 설명, 작성자 등
<meta charset="UTF-8"> 사용할 문자 설정을 표기한다.(이걸 해줘야 브라우저가 정확히 해석할 수 있다.)
<meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1"> // html4 방식이다.
<meta name="description" content="Free Web tutorials">
<meta name="keywords" content="HTML,CSS,XML,JavaScript">
<meta name="author" content="Hege Refsnes">
그 외 다양한 정보들을 닮는다.
<base href="http://www.w3schools.com/images/" target="_blank"> 루트 경로와 디폴트 타겟을 설정해준다.

url
url의 포멧: scheme://prefix.domain:port/path/filename (port는 디폴트로 http의 80을 갖는다.)
url은 아스키코드만 사용하기 때문에 다른 캐릭터셋 문자의 경우 인코딩 되어 보내진다.
